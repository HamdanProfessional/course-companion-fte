# Skill Creation Template

## SKILL.md Structure

Every skill should follow this structure:

```markdown
# [Skill Name]

**Type**: Development / Infrastructure / Testing
**Author**: [Your Name]
**Created**: [Date]

## Description
[Brief description of what this skill does]

## When to Use This Skill
Use this skill when:
- [Condition 1]
- [Condition 2]
- [Condition 3]

## Examples
See `assets/examples/` for usage examples.

## Assets
- `examples/` - Code examples showing how to use the skill
- `templates/` - Reusable templates
- `scripts/` - Helper scripts
- `references/` - External resources and best practices

## Related Skills
- [Skill 1] - Related skill
- [Skill 2] - Related skill

## Files Generated
This skill can generate:
- File type 1
- File type 2
```

## Example Skill Template

```markdown
# My Awesome Skill

**Type**: Development
**Author**: Claude
**Created**: 2025-01-01

## Description
Automates the creation of [something] for [purpose].

## When to Use This Skill
Use this skill when:
- User wants to create a new [feature]
- User needs to scaffold [component]
- User asks for help with [topic]

## How It Works
1. First, it analyzes the [context]
2. Then, it generates [output]
3. Finally, it validates [result]

## Examples
See `assets/examples/` for:
- Basic usage
- Advanced patterns
- Best practices

## Assets
- `examples/basic-usage.md` - Simple examples
- `examples/advanced.md` - Complex scenarios
- `templates/my-template.md` - Reusable template
- `scripts/generate.py` - Generation script
- `references/links.md` - External resources

## Related Skills
- backend-builder - For backend scaffolding
- frontend-component - For UI components

## Files Generated
- Component files (.tsx, .ts)
- Test files (.test.ts)
- Documentation (.md)
```
